# CH32X035G8U6 USB-SPI ブリッジファームウェア仕様書 (v1.9)

## 1. プロジェクト概要
本デバイスは、市販のUSB HIDゲームパッドを接続し、その入力を解析して本体（RP2350等）へSPI通信で高速転送する**「入力専用ブリッジ」**として動作する。

### 開発背景
RP2350にUSBゲームパッドを直結した場合、SDカードへのアクセス時にUSBホストのタイミングが乱れ、ゲームパッドが切断される事例が確認されている。切断後、最近のゲームパッドではHomeボタン操作によるDirectInputモードへの再切替が必要になるなど、ユーザー体験を著しく損なう。本ブリッジを介することで、本体側の負荷に関わらずUSB接続を維持し、安定した操作環境を実現する。

## 3. ソフトウェア実装仕様

### 3.1 SPI 通信プロトコル
汎用性と確実性を重視し、標準的な「コマンド送信 → データ受信」のフェーズ分離方式を採用する。

* **通信モード:** SPI Mode 0 ($CPOL=0, CPHA=0$)。
* **同期:** NSS（CS）の立ち下がりで内部ビットカウンタおよびポインタをリセット。
* **基本シーケンス:**
    1. **Master:** 1バイトのコマンドIDを送出。
    2. **Slave:** コマンド受信完了後、速やかに該当データを送信バッファに準備。
    3. **Master:** 必要なデータ長分、ダミークロックを送ってデータを読み出す。
* **CS制御:** 1トランザクション（コマンド＋データ完了）の間、CSをLowに保持すること。

### 3.1.1 SPIクロック周波数
* **最大:** 1MHz
* **最小:** 100kHz

### 3.2 コマンド体系

| ID (MOSI) | 名称 | 返答サイズ | 詳細 |
| :--- | :--- | :--- | :--- |
| **`0x01`** | **GET_PAD_INPUT** | 3 bytes | 解析済みの3バイト入力データ |
| **`0x02`** | **GET_PAD_RAW** | 可変 | USB生レポート（長さは `0x03` で確認） |
| **`0x03`** | **GET_PAD_RAW_LEN**| 1 byte | 現在接続中のパッドの生レポート長 |
| **`0x10`** | **GET_SYS_STATUS** | 1 byte | USB接続状態 (0:未接続, 1:接続済) |
| **`0x11`** | **GET_DEVICE_ID** | 4 bytes | 接続デバイスの VID (2B) + PID (2B) |
| **`0xFE`** | **GET_FW_VERSION** | 2 bytes | ファームウェア Ver (Major + Minor) |

## 4. SPI入力データフォーマット (3バイト固定)

`0x01` コマンドに対して送出される、CH32側で解析済みの24bitデータ。

| Byte | Bit 7 | Bit 6 | Bit 5 | Bit 4 | Bit 3 | Bit 2 | Bit 1 | Bit 0 |
| :--- | :---: | :---: | :---: | :---: | :---: | :---: | :---: | :---: |
| **0** | A | B | X | Y | L1 | R1 | L2 | R2 |
| **1** | Start | Select| L3 | R3 | 上 | 右 | 下 | 左 |
| **2** | L上 | L右 | L下 | L左 | R上 | R右 | R下 | R左 |

#### 備考
- **D-pad (上/右/下/左)**: 直交4方向のみを抽出。斜め入力（1, 3, 5, 7）は現状マッピング対象外として全ビット `0` となる。
- **アナログ閾値**: 各方向で中央（0x80）から一定以上（±64）離れた場合にビットを立てる。

## 5. 実装上の要件

### 5.1 スレーブ側 (CH32X035)
* **コマンド解析:** 1バイト目の受信完了割り込みにて、次にMISOから送り出すべきデータを動的にセットする。
* **生レポート管理:** USB HIDスタックが新しいレポートを受信した際、その長さを `0x03` 用変数へ、データを `0x02` 用バッファへ格納する。
* **排他制御（ダブルバッファリング）:** メインループによる書き込みとSPI割り込みによる読み出しの競合（ティアリング）を防ぐため、**「ロックフリー・ダブルバッファ（ピンポンバッファ）」**を実装。安定したバッファのインデックスをアトミックに切り替えることで、割り込み禁止を使わずにデータの整合性を確保している。
* **DMA転送による高精度化:** 1MHz以上の高速クロックでもジッターを抑制し安定したスレーブ応答を行うため、SPIのレスポンス送信にDMA1_Channel3を使用。
* **物理層の堅牢性:** NSS（CS）の立ち上がり時にSPIモジュールのSPEビットをトグルし、ハードウェアのビットカウンタを強制リセットすることで、外的ノイズ等による1ビットずれが後行の通信に波及しないよう自動回復制御を実装。
* **USB-C 供給制御 (CCモニタリング):** PC14/PC15のCCラインを監視。Rd(5.1kΩ)検出時に120msのデバウンスを経て給電開始。切断時は20msのデバウンスで即座に遮断するステートマシンを実装。

### 5.2 マスター側 (RP2350)
* **RAWデータ取得手順:** 必ず `0x03` で現在のレポート長を確認してから `0x02` で読み出しを行うこと。スレーブ側では読み取り順序の制限（0x03 の後に 0x02 を強制するなど）は行っておらず、単に `0x02` を呼ぶと最新のバッファが返るため、整合性を保つのはマスター側の役割となる。
